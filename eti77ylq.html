Подключение к серверу

Сюда вставляются только публичные данные клиента Supabase. Ключ service_role сюда вставлять нельзя.

Для этого проекта используйте именно Legacy API Keys → anon. Не вставляйте service_role.

Готовые ссылки

Сначала откройте главную ссылку. После этого можно копировать ссылки отделений либо отсюда, либо из главного файла.

Что делать дальше

  1. Создайте проект Supabase.
  2. Запустите SQL из supabase/migrations/20260506_create_sharsh_tables.sql.
  3. Разверните функцию из supabase/functions/sharsh-sync/index.ts.
  4. Вставьте сюда Project URL и Legacy anon key.
  5. Откройте главную ссылку и раздайте ссылки отделениям.

Если включите коды отделений на сервере, оставьте включённым пункт Показывать поле кода отделения.
